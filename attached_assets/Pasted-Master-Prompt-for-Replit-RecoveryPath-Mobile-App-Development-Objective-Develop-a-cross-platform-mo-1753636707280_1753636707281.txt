Master Prompt for Replit: RecoveryPath Mobile App Development
Objective: Develop a cross-platform mobile application called RecoveryPath for drug addiction patients in Thailand, as outlined in the provided PRD. The app should be user-friendly, visually appealing, and culturally relevant, with features for recovery tracking, educational resources, community support, crisis intervention, and gamification. Use Replit to prototype, develop, and test the app.

Environment: Replit (web-based IDE)

2. Technology Stack
Frontend: Flutter (Dart) for cross-platform iOS/Android development.
Backend: Firebase for authentication, Firestore for data storage, and Cloud Functions for server-side logic.
APIs:
Integrate with Thai helpline APIs (e.g., 1413) for crisis support (mock API if real API is unavailable).
Use Google Maps API or geolocator for locating nearby rehab centers.
Offline Support: Use cached_network_image and local storage for offline resource access.
Notifications: Use flutter_local_notifications for reminders and motivational messages.
3. Core Features Implementation
Below are guidelines for implementing the key features from the PRD in Replit:

3.1 Recovery Tracking
Objective: Allow users to log sobriety progress, mood, and medication adherence.
Implementation:
Create a Progress model in /models/progress.dart:
dart



class Progress {
  String id;
  DateTime date;
  String mood;
  bool medicationTaken;
  int sobrietyDays;

  Progress({this.id, this.date, this.mood, this.medicationTaken, this.sobrietyDays});
}
Build a TrackingScreen in /screens/tracking_screen.dart with a form for daily check-ins and a chart widget.
Use provider for state management to update progress in real-time.
Store progress in Firestore under users/{userId}/progress.
3.2 Educational Resources
Objective: Provide a library of articles and videos.
Implementation:
Create a Resource model in /models/resource.dart:
dart



class Resource {
  String id;
  String title;
  String type; // article or video
  String url;
  bool isOfflineAvailable;

  Resource({this.id, this.title, this.type, this.url, this.isOfflineAvailable});
}
Build a ResourcesScreen in /screens/resources_screen.dart with a list of ResourceCard widgets.
Cache resources using cached_network_image for offline access.
Fetch content from Firestore or a mock API (e.g., JSON file in Replit).
3.3 Community Support
Objective: Create an anonymous chat forum.
Implementation:
Use Firestore to store chat messages under chats/{threadId}/messages.
Build a CommunityScreen in /screens/community_screen.dart with a real-time chat UI using StreamBuilder.
Implement moderation logic (e.g., flag inappropriate messages) in a Cloud Function.
Ensure anonymity by using Firebase Anonymous Authentication.
3.4 Crisis Intervention
Objective: Provide one-tap access to helplines and calming exercises.
Implementation:
Build a CrisisScreen in /screens/crisis_screen.dart with buttons for:
Calling 1413 (use url_launcher to initiate calls).
Guided breathing exercises (use animations for a visual timer).
Use geolocator to display nearby rehab centers based on user location.
Store a static list of helplines in /utils/constants.dart.
3.5 Gamification
Objective: Add badges and motivational quotes.
Implementation:
Create a badge system in Firestore under users/{userId}/badges.
Display badges on the HomeScreen in /screens/home_screen.dart.
Use a local JSON file in /utils/constants.dart for Thai cultural quotes (e.g., Buddhist proverbs).
Implement weekly challenges with progress tracking.
4. UI/UX Guidelines
Theme: Create a calming theme in /utils/theme.dart with colors inspired by Thai culture (e.g., blue: #4A90E2, green: #2ECC71, gold: #FFD700).
Accessibility:
Use TextStyle with scalable fonts and high contrast.
Implement voice navigation using Flutter’s speech_to_text package.
Navigation: Use a BottomNavigationBar for easy access to Home, Tracking, Resources, Community, and Crisis screens.
Onboarding: Create an onboarding flow in main.dart with an option for anonymous login.
5. Security and Compliance
Anonymity: Use Firebase Anonymous Authentication for users who opt for no personal data.
Data Protection: Enable Firestore security rules to restrict data access:
javascript




rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth.uid == userId;
    }
  }
}
Compliance: Ensure data handling complies with Thailand’s PDPA by minimizing data collection and using encryption.
